Software Livre

O que é Open Source (Código-fonte aberto) e Software livre.

Você ouviu muitas vezes os termos código-fonte aberto o software livre. Se um programa é aberto ou não de origem, Eles não só se preocupam com os desenvolvedores, em última análise, também é importante para os usuários. O software de código aberto licenciado as liberdades de usuários que caso contrário não teria.

Código-fonte aberto é definido Se um programa de código-fonte aberto, seu código fonte está disponível gratuitamente para seus usuários. Seus usuários, e qualquer outra pessoa, Eles têm a capacidade de tirar este código-fonte, modificar e distribuir suas próprias versões do programa. Os usuários também têm a capacidade de distribuir tantas cópias do programa original como eles querem. Qualquer pessoa pode usar o programa para qualquer finalidade e não há nenhuma taxa de licença ou outras restrições sobre o software.

Assim, Nós temos o sistema operacional Ubuntu Linux que é open source. Você pode baixar o Ubuntu, criar cópias como muitos como você gosta e dar-lhes aos seus amigos. Você pode instalar o Ubuntu em um número ilimitado de computadores. Você pode criar modificações do disco de instalação do Ubuntu e distribuí-los. Se você sabe como, Você ainda pode baixar o código-fonte Ubuntu e modificá-lo, criando sua própria distribuição personalizada. Todas as licenças de código aberto permitem que você faça isso, Enquanto as restrições que fechou as licenças de fonte.

O oposto de software open source é o software de código fechado, ter uma licença que restringe os usuários e mantém o código fonte deles. O exemplo mais importante é o sistema operacional Microsoft Windows.

Diferenças entre software livre e aberto Geralmente, os aplicativos de código aberto estão disponíveis gratuitamente, Embora não há nada que impeça que as acusações de desenvolvedor para cópias do software se você permitir a redistribuição do aplicativo e seu código fonte mais tarde. No entanto, Isso não é o que preocupa o software livre. Há momentos em que a tradução da palavra grátis em software livre é tomada como livre, Quando seu significado é realmente livre, Mas isso não significa que você não tem que pagar por isso.

Tipos de licenças Existem muitas licenças diferentes utilizadas em projetos de código aberto que dependem de que os desenvolvedores querem.

O GPL, o Licença pública geral GNU, É amplamente utilizado por muitos projetos de código aberto, como o Linux. Além de todas as definições acima do open source, os termos da GPL que especifica, Se alguém modifica um programa open source e distribuir uma obra derivada, Você também deve distribuir o código-fonte para o seu trabalho derivado. Em outras palavras, Ninguém pode pegar o código e criar uma fonte fechada do programa, Mas isso deve dar suas alterações para a Comunidade. Microsoft se refere a GPL como viral por esta razão, forçando a programas que incorpora GPL código para liberar seu próprio código-fonte. É claro, os desenvolvedores de um programa podem optar por não usar que a GPL codificaria se trata de um problema.

Algumas outras licenças, como a licença BSD, Eles impõem menos restrições aos desenvolvedores. Se um programa é licenciado sob a licença BSD, qualquer pessoa pode incorporar o código-fonte do programa em outro programa. Eles não têm que dar suas alterações à Comunidade. Algumas pessoas veem que isso é mesmo mais grátis como a licença GPL, Desde que dá aos desenvolvedores a liberdade para incorporar o código em seus próprios programas de fonte fechados, enquanto outros a veem como menos livre Porque... tirou os direitos dos usuários finais do programa derivado.
